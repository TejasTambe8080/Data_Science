🧠 PART 1 — THEORY (INTERVIEW + INDUSTRY)
🔹 What is a Branch?

👉 A separate version of your code

📌 Example:

main → stable code
feature → new changes

👉 You don’t break main project while experimenting

🔹 Why Branching is Important?
Safe development
Multiple developers working together
Feature isolation
💻 PART 2 — GIT BRANCHING COMMANDS
🔹 Create Branch
git branch feature1
🔹 Switch Branch
git checkout feature1

👉 (or modern way)

git switch feature1
🔹 Create + Switch
git checkout -b feature1
🔹 Check Branches
git branch
🔹 Merge Branch
git checkout main
git merge feature1
⚠️ PART 3 — .gitignore (VERY IMPORTANT)
🔹 What is .gitignore?

👉 File that tells Git what NOT to upload

🔹 Example .gitignore
__pycache__/
*.csv
.env

👉 Why?

Avoid large files
Protect sensitive data
Clean repo
